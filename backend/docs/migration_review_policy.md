# Migration Review Policy

## Required for every migration PR
- SQLAlchemy model changes included
- Alembic migration file included
- Migration file reviewed manually
- Upgrade path described
- Rollback/downgrade path described
- Zero-downtime risk noted

## Red Flags
- Drop column/table without explicit explanation
- Add NOT NULL column on populated table in one step
- Rename column without expand-contract plan
- Large backfill mixed into schema migration without comment
- Multiple Alembic heads introduced
- Autogenerated migration committed without cleanup/review

## Preferred Style
- Small focused migrations
- Clear file names
- Explicit comments for destructive steps
- Deterministic constraint/index naming
